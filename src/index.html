Polyrepos

Definition

Polyrepos

  • one Problem = one Repo
  • one Product = many Repos
  • classic, well-understood approach

Polyrepos

Use Cases

Polyrepos

  • heterogeneous teams
  • code isolation
  • granular user access
  • heterogeneous: different coding styles, different languages, opposing teams repo structures
  • isolation: different SCMs Github vs Gitlab
  • access: e.g. grant external users access to only one part of your application

Mono · Oligo · Poly

one · few · many

Monorepos

Definition

Monorepos

  • many Problems = one Repo
  • many Products = one Repo
  • monorepo != monolith
  • relatively* new approach
  • aka: shared codebases
  • *since early 2000s1
  • monolith: distills many code fragments into one big application (e.g. a videogame)
  • monorepo: contains many individual, homogeneous (sometimes interdependent) applications (e.g. microservices)

Monorepos

Use Cases

Monorepos

  • homogeneous teams
  • D.R.Y. principle
  • holistic approach
  • can enforce company standards
  • homogeneous: share rules/coding-styles across code fragments, linting, use the same programming language
  • DRY: don't repeat yourself, write once use everywhere
  • holistic: get everything you need with every dependency traceable
  • Standards: Corporate Design, API-Interfaces, CI/CD Pipelines, Git Hooks

Monorepos

Pros

Monorepos

  • Separation of Concern
  • Config Sharing
  • Interface Sharing (API-Contracts)
  • consistent deployments
  • workspaces are installed in node_modules

Monorepos

Cons

Monorepos

  • More frequent commits (ci/cd, pulls, rebases)
  • bigger repo size
  • Access Management
  • partial clones and git sparse-checkout
